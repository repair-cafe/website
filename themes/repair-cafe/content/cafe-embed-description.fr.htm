Vous avez la possibilité d'intégrer les événements de ce Repair Café sur votre site web. Copier l'extrait de code suivant à l'endroit où vous souhaitez que les événements s'affichent sur votre site web^.

<iframe src="{iframe_url_current_cafe}" id="repairCafe" width="100%" height="400" frameborder="0"></iframe>

Structure de l'URL

{iframe_url}

Choix de la langue

La langue peut être choisie avec le paramètre URL :language. Les langues suivantes sont à disposition: {language_code_list}.

Choix du Repair Café

Changer le paramètre URL :cafe_slug permet de sélectionner le Repair Café dont les événements doivent être affichés. Le :cafe_slug peut être retiré des URL respectifs de la page du Repair Café.

Options

Nombre d’événements par page

Habituellement, {events_per_page_default} événements sont affichés par page. Pour changer le nombre, un paramètre GET events_per_page peut être ajouté à l'URL.

Exemple:
{iframe_url_current_cafe}?events_per_page=5

iFrame Resizer (optionnel)

Dans l'iFrame, le script iFrame Resizer- en version 3.6.6 a déjà été téléchargé. Pour que l'iFrame s'adapte automatiquement à la taille du contenu. Ainsi, des barres de navigation inesthétiques peuvent être évitées sur votre site web. Tout ce que vous devez faire, c'est intégrer le script iframeResizer.min.js dans votre page et l'initialiser pour l'iFrame intégré:

<script>
iFrameResize({
    checkOrigin: ['{website_origin}']
}, '#repairCafe');
</script>

Lors de l'utilisation de l'iFrame Resizer, l’attribut scrolling="no" doit être ajouté à l'iFrame. Exemple:

<iframe src="{iframe_url_current_cafe}" id="repairCafe" width="100%" height="400" frameborder="0" scrolling="no"></iframe>

Plus d'informations concernant l'intégration du script iFrame Resizer? Lien vers la documentation officielle: https://github.com/davidjbradshaw/iframe-resizer.
